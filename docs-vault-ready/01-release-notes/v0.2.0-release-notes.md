# NeuroWorks v0.2.0 â€” Release notes

> The AI Workforce now hires teams. Multi-persona dispatch, document context,

## Headline features

### Multi-persona team dispatch â€” `/team`
Dispatch a single brief to N employees in parallel. Each one runs as its own
job through the same `planAndExecute` pipeline as chat, with its persona
suffix pinned at dispatch time so the planning bias holds even when the
global active persona changes underneath. Capped at 12 tasks per call.

**Page:** `/team` (left nav, between Chat and Tasks)
**API:** `POST /api/team` with `{ tasks: [{ persona?, content, attachments? }] }`
**Routing:** each task auto-fans across primary + worker peers; an extra
worker auto-spawns when peer inflight â‰Ą 1.

### Document uploads with context-aware folding
Two upload targets:

- **context** â€” staged for THIS chat / team dispatch only, TTL 1 h, text auto-extracted from PDF / DOCX / XLSX and folded into the planner prompt as primary evidence.
- **vault** â€” permanent import into the knowledge vault (default folder `0-Inbox`).

**UI:** paperclip button + target toggle in the Chat input row; "Attach a
document" button on the Team page applies the doc to every employee in the
brief.

**API:** `POST /api/uploads` with `{ filename, contentBase64, target,
mimeType? }`.

### Persona auto-router
When no specialised persona is active (or the generalist clawbot is on),
chat tasks now auto-route to the right specialist via a regex pattern
catalog covering all 17 built-in personas. The router only fires when one
persona is a confident single winner â€” ties fall through to clawbot so we
never guess.

**Surfaced as** `personaAutoRouted: { personaId, score, matched }` on the
chat response and stamped on `job.inputs`.

**Coverage:** account-executive, marketing-manager, customer-success,
software-engineer, devops-sre, qa-engineer, product-manager,
product-designer, financial-analyst, operations-coordinator, recruiter,
contracts-reviewer, data-analyst, technical-writer, executive-assistant,
researcher, plus the generalist clawbot fall-through.

### Past-3-sessions resume
The Chat header now shows a `Recent` strip with chips for your last three
sessions. Click any chip to swap that thread back into view. Snapshots
happen automatically after every exchange â€” no "save session" required.

### Review-driven retry loop
When `peer.review` returns `needs-work` or `bad` after the existing
quality-rescue and OR-rescue paths, the agent now:

1. Re-synthesises with the reviewer's specific issues as explicit revision
   instructions.
2. Re-runs `peer.review` on the retry draft.
3. If the retry clears (`verdict: good`) â†’ swaps the draft **and** the
   surfaced review so the user only sees the final clean verdict.
4. If still flagged â†’ compares quality scores and keeps whichever is better.

End result: customers no longer see `Peer review: needs-work / "Contact
details inconsistent" / "Minor filler"` chips when the system can correct
itself.
