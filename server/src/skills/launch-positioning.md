# Skill: Launch positioning

## Goal

A short customer-facing announcement that an existing user reads and thinks: "Yes — that solves a real thing for me." NOT a press release. NOT a feature dump.

## Length guide

| Surface | Length |
|---|---|
| Changelog entry | 30-60 words |
| Launch blurb / one-liner | 1-3 sentences |
| Customer email subject + body | subject 6-10 words; body 100-180 words |
| Status / "what's new" banner | 1 sentence under 18 words |

## Shape (in order)

1. **Hook — the benefit, in the customer's words.** Lead with what they CAN now do that they couldn't before. ("Export every dashboard in a workspace as a single zip.") No "we're excited to announce".
2. **Why it matters — one line.** The pain it removes or the time it saves. Be specific: "no more 40-click exports".
3. **What's new — concrete.** What changed. Reference the feature by the name the user sees in the UI.
4. **CTA (if any).** Where to go. Linked in product surfaces, not "contact support".

## Template

```
**<Hook sentence — capability, not implementation.>**

<Why it matters — pain removed, time saved, friction gone.>

<One-line "how": where in the product, what it looks like.>

<CTA — link to the actual surface, not the docs index.>
```

## Rules

- **Benefit, never implementation.** "Reports load 3× faster" beats "we migrated to a new query engine".
- **Customer's vocabulary, not internal.** Strip codenames, internal feature flags, team names.
- **No "we're thrilled".** Excitement belongs in the press release, not the changelog.
- **Strip "revolutionary / game-changing / next-gen / industry-leading".** Vendor speak.
- **Quantify when honest.** "3× faster" is good. "Significantly faster" is filler.
- **One thing per launch.** If it's a bundle, lead with the most material change and link to the full notes.

## Pitfalls

- **Implementation in the hook.** Nobody buys "we migrated to a new query engine". They buy "reports load 3× faster".
- **Wall of bullets.** If you have 5 things to announce, you have 5 launches. Pick the biggest.
- **Marketing puffery.** "Best-in-class", "industry-leading" — replace with the specific number that proves the claim. If you can't, drop the claim.
- **CTA to nowhere.** "Check it out" without a link to the surface. The CTA should land the user IN the feature.
- **Apologetic intro.** "After many months of work" — the customer doesn't care; lead with the result.
