# Skill: Logistics exception resolution

## Goal

A shipment went wrong — contain the customer impact fast, drive the fix, keep stakeholders informed, and close with a root cause so it doesn't recur.

## Structure

```
# Exception — <ref> · <type: delay | damage | lost | customs hold | shortage | mis-ship>
Detected: <when/how> · Customer impact: <what they feel + by when>

## 1. Assess
- What happened, where in the chain, scope (one order / batch / lane)
- Severity: <critical (customer down) / high / medium>
- Who's affected + the deadline that's now at risk

## 2. Contain (stop the bleeding)
- <expedite alt shipment, partial ship, source from another DC, hold further dispatches on this lane>

## 3. Resolve
| Action | Owner | By |
|---|---|---|
| <e.g. engage carrier / broker / insurer> | | |

## 4. Notify
- Customer: <what we say — honest status + new ETA + what we're doing> (see crisis-comms for tone)
- Internal: <sales/CS/finance who need to know>

## 5. Root cause + prevention
- 5-whys → <true cause: carrier SLA / packing / docs / forecast / supplier>
- Preventive action + owner + due
- Claim filed? <insurer/carrier ref, amount>
```

## Rules

- **Contain before you perfect the fix.** A partial shipment or expedited replacement today beats the ideal solution next week.
- **Quantify customer impact and the at-risk deadline first** — that sets the urgency and the comms.
- **Communicate proactively with a new ETA** — silence turns a logistics problem into a relationship problem (use the crisis-comms tone for serious cases).
- **File carrier/insurance claims promptly** — claim windows are short; gather evidence (photos, POD, packing) now.
- **Always close with a root cause + preventive action + owner** — an exception with no prevention is a recurring cost.

## Pitfalls

- Chasing the perfect fix while the customer sits blind — contain and inform first.
- No new ETA in the customer message — "we're looking into it" erodes trust.
- Missing the claim window — the cost lands on you.
- Closing the ticket without a root cause — the same lane fails again next month.
