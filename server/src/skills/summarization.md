# Skill: Summarization

## Goal

The reader skips the original, gets the same decisions / actions / facts in 1/10th the time.

## Output shape

```
**Bottom line:** <one sentence — the single thing the reader needs to know>

## Key points
- <Fact / decision / action — most important first> [source]
- <…>
- <…>

(Optional) ## Quotes worth keeping
> "<exact quote that matters>" — <attribution>

## Sources
- [Title](url) or [vault:path/to/note.md]
```

## Principles

- **Bottom line first.** If the reader only gets one line, what is it?
- **Lead with what the source DECIDED or RECOMMENDED**, not what it described.
- **3-7 bullets.** Less and you're under-summarizing; more and you're paraphrasing.
- **Lose tone, keep stance.** "The author argues X" stays in; "the author elegantly demonstrates that…" goes.
- **Numbers stay verbatim.** "Revenue up 23%" stays as 23%, not "significantly up".
- **Each bullet stands alone.** Don't write "and also" / "moreover" — bullets aren't a paragraph.

## Length cap

- Short doc (< 1k words): 80-150 word summary.
- Long doc / multi-source: 150-300 word summary + a Sources block.
- Anything longer than 300 words isn't a summary — it's an abridgement. Split it.

## Pitfalls

- Burying the lede ("The article begins by discussing…" — get to the point).
- Inventing structure the original didn't have (don't impose Pros/Cons if the source isn't one).
- Dropping numbers and dates ("recently" instead of "Q3 2026").
- Adding your own opinion. The summary mirrors the source; your take goes in a separate section.
