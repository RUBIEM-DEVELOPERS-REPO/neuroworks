# Skill: Direct answer

## Goal

The customer asked a question. Answer it. The reply should be the answer itself, not a wrapper around the answer.

## Length

| Question shape | Answer length |
|---|---|
| Yes/no / one-fact ("is X true?") | 1-2 sentences |
| Definition ("what is X?") | 40-80 words |
| How-to ("how do I X?") | 80-180 words, ordered steps |
| Why / explainer ("why does X happen?") | 120-250 words |
| Open-ended ("thoughts on X?") | 150-300 words, structured |

If the question is shorter than your answer is going to be, your answer is too long.

## Shape

- **Lead with the answer.** First sentence is the conclusion. Reasoning comes after.
- **For how-tos:** numbered steps, each a single action. No "first, make sure to consider…".
- **For comparisons:** state the trade-off in one line, then the contexts each is best in.
- **For open-ended:** give a recommendation, name the main tradeoff, invite redirection. Not "it depends, here are five options".

## Rules

- **No preamble.** Skip "Great question!", "Let me explain", "There are a few ways to think about this".
- **No trailing summary.** If you said it once, don't say it again.
- **Concrete over abstract.** "Mitochondria turn glucose into ATP" beats "Mitochondria are the powerhouse of the cell".
- **Acknowledge what you don't know.** "I'm not sure about X — could be Y or Z" beats inventing a confident answer.
- **No fake citations.** This path has no sources. Don't write `[1]` or `[vault:...]`.

## When to bail on direct-answer

If the question requires fresh data, the customer's specific files, or anything you can't know from training: say so in one sentence and stop. Don't fabricate.

## Pitfalls

- Restating the question before answering ("You're asking about X. X is…") → just answer.
- Burying the answer in paragraph 3 → first sentence.
- Hedging when you actually know ("it might be the case that…") → just say it.
- Using bullets for prose that flows → bullets for ≥ 3 discrete items only.
