# Skill: Decline politely

## Goal

The operator needs to say no without burning the relationship. The reply
is short, clear, and doesn't open a negotiation the operator wants closed.

## When to use which shape

| Situation | Shape |
|---|---|
| Meeting that doesn't need the operator | Soft — suggest delegate |
| Recurring meeting drained of value | Soft — propose async alternative |
| Pitch / sales ask the operator doesn't want | Hard — clear, no-negotiation |
| Favour ask that's not aligned with current priorities | Hard with empathy |
| Vendor cold pitch | Hard, one-liner |
| Internal teammate asking for an unfair time commit | Soft — counter-offer |

## Process

1. **Decide hard or soft.** Soft = "no, but here's a useful alternative".
   Hard = "no, and please don't follow up".
2. **Acknowledge the ask.** One line — shows you understood, not
   dismissive. Don't apologise excessively.
3. **State the no.** Clear, in the FIRST sentence after the
   acknowledgement.
4. **Optional — give one reason.** Not justification, just context.
   Examples: "in a quiet period this quarter", "out of remit", "we use
   a different vendor".
5. **Optional — offer an alternative.** A counter-time, a delegate, an
   async update, a different vendor.
6. **Close warmly but firmly.** No "let me know if anything changes" —
   that opens the loop again.

## Output — soft shape

```
Hi <Name>,

Thanks for thinking of me / for the invite.

I'm going to skip this one — <one-line reason if useful>. <delegate /
async alternative / counter-offer>.

Appreciate the heads-up.

— <Operator>
```

## Output — hard shape

```
Hi <Name>,

Thanks for reaching out.

This isn't something we're pursuing right now. <one-line reason>.

Wishing you well with it.

— <Operator>
```

## Output — meeting decline

```
Hi <Name>,

Going to skip this one — <delegate's name will join / async update
is fine for me>. Happy to read the notes after.

— <Operator>
```

## Rules

- **No is in the first 20 words.** Burying it makes the reader read
  twice and feel manipulated.
- **No "let me know if anything changes" on hard nos.** That sentence
  invites another pitch in three months.
- **Don't lie about the reason.** "Busy quarter" is fine; "we're doing
  X already" is fine if true. Inventing a reason is a trust tax.
- **Empathy is one sentence, not three.** Padding empathy makes the
  decline read insincere.

## Pitfalls

- Apologising too much ("So sorry, but…"). Apologies read as soft yes.
- Hedging ("for now / not at the moment / maybe later") when the answer
  is no for the foreseeable future. Lying.
- Long explanations. The longer the reason, the weaker the no.
- Suggesting a follow-up when there's no real reason for one. Polite
  theatre.
