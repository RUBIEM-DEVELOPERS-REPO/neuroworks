# Skill: List making

## Goal

A list the reader can scan in 10 seconds and act on. Items that read like a paragraph chopped into bullets is a failed list.

## Rules

1. **One grain.** Every item is the same kind of thing. "Buy milk", "go to the gym", "rethink my career" doesn't belong in one list.
2. **Parallel grammar.** All verbs, or all noun phrases, or all questions. Mixing breaks scanning.
3. **Order matters.** Sort by priority, dependency, or time — whichever serves the reader. Random-order lists waste the reader's eyes.
4. **No filler items.** If an item exists only to round the list to 5, cut it.
5. **Sublists only when nesting is real.** Don't sub-bullet for visual decoration.
6. **3-9 items.** Two items aren't a list — write a sentence. Ten+ items want grouping (use a table or sections).

## When to use each shape

- **Checklist (☐):** the reader will tick items off as they do them.
- **Bullets (•):** discrete items the reader scans but doesn't act on linearly.
- **Numbered (1.):** order is load-bearing — steps, ranks, dependencies.

## Output shape

For a step-by-step task:
```
1. <Action verb> <object> — <expected result>
2. ...
```

For a brainstorm or option list:
```
- <Option> — <one-line why / when>
- <Option> — <one-line why / when>
```

For a checklist:
```
- [ ] <Action — owner if multi-person>
- [ ] <Action>
```

## Pitfalls

- Bulleting paragraphs of prose → no scanning value. Either tighten to one-liners or use a paragraph.
- "And so on" / "etc." at the end → either cut the item or name it specifically.
- A list of vague nouns ("Communication", "Strategy", "Process") with no verbs — looks like a slide deck section, not a list.
- Items longer than 2 lines each → the list wants to be a table or sections.
