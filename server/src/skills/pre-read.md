# Skill: Pre-read

## Goal

Every attendee opens the pre-read once, skims for 60 seconds, and walks
into the meeting knowing (a) what's being discussed, (b) what the
proposing party recommends, (c) what they're being asked to decide. No
surprises, no "wait what's the context here" question at minute 10.

## Process

1. **State the situation in ≤3 lines.** What changed, when, and why
   we're meeting now.
2. **State the options on the table.** Usually 2-3. Each option needs:
   - One-line description
   - One-line on the upside
   - One-line on the cost / risk
3. **State the recommendation.** The proposing party SHOULD have a view.
   Hiding behind "we want to discuss" wastes the room.
4. **State what we need from the room.** Approval? A decision between
   options? Input that informs the proposer's decision? Be explicit.
5. **List who needs to give input + by when.** Names + deadline. No
   floating action items.

## Output shape

```
# Pre-read — <Topic> · <Meeting date>

## Situation
<2-3 lines: what changed, deadline, why we're meeting>

## Options

### Option A — <one-line label>
- **What it is:** <one line>
- **Upside:** <one line>
- **Cost / risk:** <one line>
- **What it'll cost:** <effort, $, time, opportunity cost>

### Option B — <one-line label>
- **What it is:** <…>
- **Upside:** <…>
- **Cost / risk:** <…>
- **What it'll cost:** <…>

### Option C — <one-line label> (optional)
- <…>

## Recommendation
> <One sentence stating which option and why>

## What we need from the room
- **Decision:** <approve A / pick A vs B / give input>
- **By when:** <date>
- **From whom:** <names — who needs to sign off>

## Open questions for the room (3-5)
- <Question that the recommendation can't answer alone>
```

## Rules

- **Under 400 words.** Pre-reads that don't fit on one screen don't get
  read.
- **Don't bury the recommendation.** Top-third of the document.
- **Cost ≠ price.** Include team-time, opportunity cost, and political
  cost where relevant.
- **Options are mutually exclusive.** If A + B can both happen, they're
  not options — they're a plan.

## Pitfalls

- Three pages of background. Background is a footnote — link to the
  source doc.
- "We'll figure it out in the meeting" — the WHOLE POINT is to give
  attendees the data to come prepared.
- Padding the recommendation with hedges ("we lean toward A but…").
  State A, name the trade-off, move on.
