# NeuroWorks progress — 2026-05-24

Today's work landed and is pushed to `origin/claude/enhance-clawbot-tools-vt8fH` as commit `402a141`.

## Features shipped

### Document uploads
`POST /api/uploads` accepts JSON-base64 with two targets:
- `context` — staged under `.neuroworks/context-uploads/<contextId>__<filename>`, TTL 1 h, extracted text returned via `resolveContextAttachment(contextId)`
- `vault` — imported via `importBinaryIntoVault` into `0-Inbox/` (or caller-specified folder)

Chat `POST /api/chat` now accepts `attachments: [{contextId}]` and folds each attachment's extracted text into the enriched task with explicit "primary evidence" framing. Per-attachment chip is removable in the UI; job log records each attachment with filename + char count so you can verify the planner saw it.

UI: paperclip button + context/vault toggle in the Chat input row. 20 MB decoded cap; PDF / DOCX / XLSX auto-extract via `extractDocText`.

### Persona auto-router
`lib/persona-router.ts` — regex pattern catalog covering all 17 built-in personas (AE, marketing, CSM, SWE, SRE, QA, PM, designer, finanalyst, ops, recruiter, contracts, dataanalyst, techwriter, EA, researcher). When `getActivePersona()` is null or returns the generalist `clawbot`, the chat handler calls `autoRoutePersona(text)` — confident single-winner pattern matches route the task to a specialised persona; ties fall through to clawbot.

`personaAutoRouted` is surfaced in the response and stamped on `job.inputs` + `job.log`.

PoC: 6 / 6 routed correctly at A grade.

### Multi-persona Team page (`/team`)
New endpoint `POST /api/team` accepts `{ tasks: [{persona?, content, attachments?}] }`. Each task dispatches in parallel through the same `planAndExecute` pipeline as chat, with `personaSystemSuffix` pinned at dispatch time (no global active-persona race). Capped at 12 tasks per call.

New page `/team`: shared brief + per-role instructions for each assigned employee, attach context docs once for all of them, dispatch fires all jobs in parallel. Live polling, status pills, per-employee answer, link to full result page.

Sub-agents naturally spawn — each team job runs through the normal peer-routing path, so a 5-employee dispatch fans out across primary + worker(s); auto-spawn-worker triggers when peer inflight ≥ 1.

PoC: team dispatch grade A, 4 / 4 per-job answers grade A, 97 s end-to-end for 4 personas.

### Past-3-sessions resume in Chat
`Recent` chip strip appears under the chat header when there's any other session in the local buffer. Auto-snapshots after every exchange (debounced via React's natural batching) and when "New session" is clicked. Click any chip → that session's full message thread swaps back into view. Capped at 3, stored in `neuroworks.chat.recent` in localStorage.

### Review-driven retry loop in `agent.ts`
When `peer.review` returns verdict `needs-work` / `bad` after the existing quality-rescue and OR-rescue paths, the agent now:
1. Re-synthesises the answer with the reviewer's specific issues injected as explicit "revision instructions" — addressing each named issue
2. Re-runs `peer.review` on the retry draft
3. If the retry's verdict is `good` → swaps both the draft AND the surfaced review object so the user sees the clean verdict, not the failed first attempt
4. If still not good → compares quality scores and keeps whichever is better

User no longer sees `Peer review: needs-work / "Contact details inconsistent" / "Minor filler"` chips in the UI when the system can correct itself.
